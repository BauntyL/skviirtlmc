# Исправления ошибки синхронизации плагина

## Проблема
Плагин SkviirtlWebConnector не мог связаться с сайтом и возвращал ошибку 500:
```
[SkviirtlWebConnector] Sync API returned code: 500
[SkviirtlWebConnector] Error details: {"message":"Internal Server Error"}
```

## Что было исправлено

### 1. Добавлено детальное логирование
- Логирование всех входящих запросов с информацией о методе, URL, body и query параметрах
- Логирование каждого этапа синхронизации (статистика сервера, игроки, кланы)
- Детальное логирование ошибок с stack trace

### 2. Улучшена обработка ошибок
- Каждая секция синхронизации теперь обернута в try-catch
- Ошибки в одной секции не прерывают обработку других секций
- Ошибки при обработке отдельных игроков/кланов не прерывают весь процесс

### 3. Добавлены проверки типов
- Проверка типа данных перед вызовом `.replace()` на полях кланов
- Защита от ошибок при обработке некорректных данных

### 4. Улучшена обработка кланов
- Безопасное удаление кланов с обработкой ошибок
- Проверка типов для имен кланов перед обработкой

## Как развернуть исправления

### Быстрый способ (рекомендуется)

1. Подключитесь к VPS:
```bash
ssh root@skviirtl-server.ru
```

2. Остановите сервер:
```bash
cd /var/www/skviirtl-site
pm2 stop skviirtl-site
```

3. Создайте резервную копию:
```bash
cp -r /var/www/skviirtl-site /var/www/skviirtl-site.backup.$(date +%Y%m%d_%H%M%S)
```

4. Загрузите новые файлы (с вашего компьютера):
```bash
# Из папки AAAsite/SkviirtlServerSite
scp -r dist/ root@skviirtl-server.ru:/var/www/skviirtl-site/
scp -r server/ root@skviirtl-server.ru:/var/www/skviirtl-site/
```

5. Запустите сервер:
```bash
pm2 start skviirtl-site
pm2 save
```

### Полная пересборка (если быстрый способ не помог)

1. Соберите проект локально:
```bash
cd AAAsite/SkviirtlServerSite
npm run build
```

2. Создайте архив:
```bash
zip -r deploy.zip dist/ server/ shared/ api/ package.json ecosystem.config.cjs
```

3. Загрузите на сервер:
```bash
scp deploy.zip root@skviirtl-server.ru:/var/www/
```

4. На сервере:
```bash
ssh root@skviirtl-server.ru
cd /var/www/skviirtl-site
pm2 stop skviirtl-site

# Резервная копия
cp -r /var/www/skviirtl-site /var/www/skviirtl-site.backup.$(date +%Y%m%d_%H%M%S)

# Распаковка
unzip -o /var/www/deploy.zip

# Запуск
pm2 start skviirtl-site
pm2 save
```

## Проверка работы

### 1. Проверьте логи сервера
```bash
pm2 logs skviirtl-site --lines 50
```

Вы должны увидеть:
```
[PLUGIN] Request received: { method: 'POST', url: '/api/sync?...', hasBody: true, ... }
[PLUGIN] Action: sync
[PLUGIN] Sync completed successfully
POST /api/sync 200 in XXXms
```

### 2. Проверьте логи Minecraft сервера
После следующей синхронизации (каждые 60 секунд) вы должны увидеть:
```
[SkviirtlWebConnector] Successfully synced data to website. Code: 200
```

### 3. Тестирование API вручную
```bash
curl -X POST "https://skviirtl-server.ru/api/sync?secret=skviirtl_secret_key_123&action=sync" \
  -H "Content-Type: application/json" \
  -d '{"secret":"skviirtl_secret_key_123","onlineCount":1,"maxPlayers":100,"tps":"20.0","players":[],"clans":[]}'
```

Ожидаемый ответ:
```json
{"status":"synced"}
```

## Что делать, если проблема не решена

1. Проверьте логи на наличие конкретных ошибок:
```bash
pm2 logs skviirtl-site --err --lines 100
```

2. Проверьте, что база данных доступна:
```bash
# В логах должно быть подключение к PostgreSQL
pm2 logs skviirtl-site | grep -i "database\|postgres"
```

3. Проверьте переменные окружения:
```bash
cat /var/www/skviirtl-site/.env
```

Должны быть:
- `DATABASE_URL` - строка подключения к PostgreSQL
- `API_KEY` - ключ API (должен совпадать с плагином)

4. Если ничего не помогло, откатитесь к резервной копии:
```bash
pm2 stop skviirtl-site
rm -rf /var/www/skviirtl-site
cp -r /var/www/skviirtl-site.backup.YYYYMMDD_HHMMSS /var/www/skviirtl-site
cd /var/www/skviirtl-site
pm2 start skviirtl-site
```

## Дополнительная информация

- Логи теперь содержат подробную информацию о каждом этапе синхронизации
- Ошибки в одной части синхронизации не влияют на другие части
- Все изменения обратно совместимы с текущей версией плагина
